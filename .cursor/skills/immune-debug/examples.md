# 黄金输出示例

## 示例 1：稳定正式防护

```markdown
## 调试摘要

触发条件:
- 选择配送方式后再触发优惠券重算时失败
- 通过既有结账集成测试可稳定复现

证据:
- 集成测试在结账状态流转阶段失败
- 日志显示重算逻辑重复进入同一状态分支
- 正常路径中该分支应只进入一次

根因:
- 结账状态机缺少重算阶段的状态流转保护
- 问题根因是非法重复流转，不是测试环境波动

修复动作:
- 为重算前的状态迁移增加 guard
- 保证同一轮结账流程不会重复进入冲突分支

验证:
- 原失败集成测试现在通过
- 相关结账测试集全部通过
- 手动回放同一路径时不再出现重复流转日志

## 免疫决策

主决策:
- 测试

决策理由:
- 该问题具备复发可能
- 预期行为可以被稳定断言

附带动作:
- 增加一条简短的 repo 级文档备注

作用范围:
- 仓库

路由:
- 正式区

置信度:
- 高

## 后续动作
- 仓库上下文: 无
- 资产写入: 正式区
- 人工复核: 否
```

## 示例 2：广域候选防护

```markdown
## 调试摘要

触发条件:
- 异步草稿发布时，多个团队会绕过同一个领域不变量
- 相同类型失败出现在同一业务域的多个仓库中

证据:
- 多个仓库的事故日志都显示相同的不变量违规
- 当前仓库修复后，类似模式仍会在别处复发

根因:
- 这类失败本质上是重复性的判断和流程失误
- 根因已经超出单仓库实现层面

修复动作:
- 修复当前仓库事故
- 将更广域的防护先放入候选区，而不是直接转正

验证:
- 当前仓库问题已修复
- 但广域规则仍未在 domain 范围内验证稳定性

## 免疫决策

主决策:
- 提示词/规则

决策理由:
- 模式真实且代价较高
- 过早激活广域规则可能导致过拟合

附带动作:
- 继续收集类似事故，供后续提升审查

作用范围:
- 领域

路由:
- 候选区

置信度:
- 中

## 后续动作
- 仓库上下文: sync-context
- 资产写入: 候选区
- 人工复核: 是
```
