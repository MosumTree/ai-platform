# 这是一份将“系统化排障”升级为“组织级免疫系统”的 Skill 开发蓝图。

对于掌舵着数千万行代码、跨越数百个前端工程仓库的基建体系而言，个人的 Debug 能力是无法规模化的。真正的解法，必须回归到工程的本质。正如那个核心公式所言：**Scaffolding + Harness = 超强生产力**。

这个被命名为 `immune-debug`（免疫调试）的 Skill 方案，正是要让 AI 成为 Harness（安全套件/测试线束）的自动构建者。

---

### 🧬 Skill 蓝图：`immune-debug`

#### 一、 核心元指令 (Meta-Prompt Constraints)
在 Skill 的入口，必须确立 AI 的行为底线，剥夺其“自由发挥”的权力：
* **无证据不施救**：禁止基于猜测修改代码，必须通过日志或测试重现。
* **面向防腐层编程**：修复不仅仅是让当前测试变绿，必须思考是否能沉淀为底层架构（Scaffolding）的约束。
* **闭环沉淀法则**：严禁在未输出“免疫规则（Linter/Test/Rule）”的情况下结束工作流。

#### 二、 扩展的工作流阶段 (The 6-Phase Pipeline)
在 `systematic-debugging` 原有四大阶段的基础之上，向两端延伸，构建完整的免疫闭环：

**▶ Phase 0: 领域感知与知识检索 (Domain-Driven Retrieval)**
* **触发机制**：读取报错信息及当前文件所在的目录结构。
* **动作**：识别当前的限界上下文（Bounded Context，例如是订单域还是用户中心），并隐式加载该领域专属的排障规则与上下文索引。
* **约束**：优先比对历史高频错误图谱。如果命中，直接提取历史根因与避坑指南，进入 Phase 3（假设验证）；未命中则进入常规溯源。

**▶ Phase 1 & 2: 现场勘探与模式分析 (Investigation & Analysis)**
* **动作**：严格执行 `systematic-debugging` 的核心流程。拉取堆栈追踪，分析代码模式，理解架构意图。
* **约束**：强制隔离上下文。复杂排查必须 Fork 子代理去处理，保持主控 Prompt 的干净。

**▶ Phase 3 & 4: 科学验证与实施修复 (Hypothesis & Implementation)**
* **动作**：提出假设 → 编写最小化测试 → 实施修复。
* **约束**：修复代码必须在隔离的沙箱或通过本地的 CI 检查链（Type Check, Lint, Test）后才能提交。

**▶ Phase 5: 验尸与快照提取 (Post-mortem Snapshot)**
* **动作**：Bug 修复且测试通过后，触发结构化总结引擎。
* **输出格式**：
  * `[Trigger]`: 触发该错误的极简代码片段或边界条件。
  * `[Root Cause]`: 框架层或业务层的根本冲突点。
  * `[Resolution]`: 标准解法。

**▶ Phase 6: 免疫转化 (Immune Translation)**
* **动作**：这是决定该方案是否具备“顶级优雅”的关键。AI 必须将 Phase 5 的总结转化为机器可读的 Harness。
* **分级转化策略**：
  1. **代码规范级**：将经验沉淀为 prompt/rule 或共享约束。
  2. **静态分析级**：为高频前端框架陷阱生成静态检测规则草案。
  3. **测试线束级**：将重现该 Bug 的路径转化为永久存在的 E2E 或单元测试用例。

### 🚀 落地演进路线图 (Rollout Strategy)

在一个数百人的大型前端团队中推行此 Skill，建议采用渐进式的三步走策略：

#### V1.0：轻量级外脑（Markdown + RAG）
* **开发重点**：实现上下文检索和结构化记录。
* **形态**：在单体仓库或核心模块中引入。排障结束后，AI 自动在项目上下文中沉淀经验。
* **价值**：跑通流程，收集第一批高质量的“血泪教训”，教育 AI 习惯这种闭环工作流。

#### V2.0：领域驱动与上下文精准投喂
* **开发重点**：完善限界上下文与共享参考模式的识别。
* **形态**：针对海量仓库建立中心化经验索引。AI 在介入排障前，拉取相关模块与历史沉淀。
* **价值**：解决信息过载，避免全局规则跨域误杀，提升调试命中率。

#### V3.0：绝对防御（The Ultimate Harness）
* **开发重点**：打通静态规则生成与自动化测试回写。
* **形态**：当 AI 识别出某种坏味道或架构缺陷引发的 Bug 时，不只生成文档，而是直接形成带新规则和失败测试的变更。
* **价值**：真正实现“无人掉进同一条河两次”。基础设施通过 AI 的日常排障完成自我进化。
